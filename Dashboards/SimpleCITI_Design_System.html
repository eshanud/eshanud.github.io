P5Do / Don't

Usage Guidelines

Key rules for maintaining visual and functional consistency across CapiCITI screens.

✓ Do
Use --navy for all sidebar, table header, and KPI card backgrounds. It's the primary brand surface.
✕ Don't
Use navy for content cards or panels. White (#fff) is always the surface for interactive content areas.
✓ Do
Use JetBrains Mono for ALL financial figures — LTV, loan amounts, DSCR, interest rates. Every number a loan officer acts on.
✕ Don't
Mix DM Sans and JetBrains Mono for the same type of data. Monospaced numerals are a deliberate trust signal — be consistent.
✓ Do
Use the stage pill system consistently. Each stage has one fixed color — never use amber for Approved or green for Underwriting.
✕ Don't
Create new pill colors for edge cases. Use the Flagged (red) pill for any deal with a blocker, regardless of its current stage.
✓ Do
Use --gold sparingly — logo, the CITI mark, key highlights, and equity in the capital stack. It's a brand accent, not a general purpose color.
✕ Don't
Use gold for interactive elements, buttons, or status indicators. Those roles belong to --accent blue and the semantic colors.
✓ Do
Surface LTV at row level in the pipeline table. It must be visible without a click — it's the first metric a loan officer checks.
✕ Don't
Hide LTV or any primary risk signal behind a drill-down. If it requires a click to find, it's creating overhead — not reducing it.
